# 无损记忆系统使用指南

## 系统架构

```
用户消息 → XiaoYi Server → OpenClaw Gateway → Lossless-Claw
                                     ↓
                           DAG 摘要压缩 + FTS5 搜索
                                     ↓
                              AI 回复
```

## 启动服务

```bash
# 一键启动所有服务
Start-All-Services.bat

# 或分步启动
# 1. 启动 OpenClaw 网关
openclaw gateway

# 2. 启动小易服务
node server-with-openclaw.js

# 3. 访问
# http://localhost:3000/voice.html
```

## 记忆功能说明

### Lossless-Claw 无损记忆

**核心特性**：
- **DAG 摘要树** - 消息→叶子摘要→中间摘要→根摘要，层级压缩
- **无损压缩** - 所有原始消息都保留，可随时展开查看
- **CJK 优化** - 正确估算中文 token 数量
- **智能工具** - AI 可使用 `lcm_grep`、`lcm_describe`、`lcm_expand`

**工作原理**：
1. 每条消息都被持久化到 SQLite 数据库
2. 当对话超过阈值时，自动压缩为摘要
3. 摘要形成 DAG 树结构
4. AI 可通过工具搜索和展开历史

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `freshTailCount` | 32 | 保护最近 N 条消息不被压缩 |
| `contextThreshold` | 0.75 | 上下文使用率达到此值触发压缩 |
| `incrementalMaxDepth` | -1 | 压缩层级深度（-1 无限） |
| `summaryModel` | - | 用于生成摘要的模型 |

## API 使用

### 对话接口

```javascript
// POST /api/chat
{
  "message": "你好，记住我喜欢喝咖啡",
  "sessionId": "user-123"
}

// 响应
{
  "success": true,
  "message": "好的，我记住了你喜欢喝咖啡！",
  "provider": "openclaw",
  "memory": "lossless-claw"
}
```

### 搜索历史

```javascript
// GET /api/memory/search?query=咖啡
{
  "success": true,
  "results": [
    {
      "content": "用户说喜欢喝咖啡",
      "timestamp": "2026-04-11T10:00:00Z",
      "sessionId": "user-123"
    }
  ]
}
```

## 记忆关键词

当用户消息包含以下关键词时，系统会特别标记为重要信息：

- `记住`、`记住我`
- `我喜欢`、`我讨厌`
- `我的名字`、`我的生日`
- `重要`、`关键`
- `别忘了`、`记得`
- `偏好`、`习惯`

## 对比：三种记忆方案

| 特性 | 内存历史 | ContextEngine | Lossless-Claw |
|------|----------|---------------|---------------|
| 持久化 | ❌ 重启丢失 | ✅ SQLite | ✅ SQLite |
| 容量限制 | ~10条 | ~50条会话 + ~1000长期 | 无限 |
| 压缩方式 | 截断 | 滑动窗口 | DAG摘要 |
| 搜索能力 | ❌ | 向量搜索 | FTS5全文搜索 |
| 展开历史 | ❌ | ❌ | ✅ lcm_expand |
| 中文优化 | 普通 | 普通 | **优化** |

## 故障排除

### OpenClaw 网关未运行

```
错误: OpenClaw 网关未运行
解决: 运行 start-openclaw-gateway.bat 或 openclaw gateway
```

### 记忆不生效

1. 检查 OpenClaw 配置中 `contextEngine` 是否为 `lossless-claw`
2. 确认 `OPENCLAW_ENABLED=true` 在 `.env` 中
3. 重启服务
