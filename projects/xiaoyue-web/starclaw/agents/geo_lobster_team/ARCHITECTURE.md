# GEO龙虾 - 生成式引擎优化智能体团队

## 核心理念

从"外包SEO服务"到"企业内生AI流量官"的范式转移：
- **传统模式**: 企业 → 外包SEO → 等待排名 → 被动获客
- **GEO龙虾模式**: AI感知引擎 → 自主决策 → 多平台内容生成 → 实时优化 → 自我进化

**定位**: 企业内生的、可7x24小时工作的"首席AI流量官"

---

## 一、环境感知与数据内化层（感官与记忆）

### 智能体1.1: GEO感知雷达 - `geo_scout`

**职责**: 全域AI搜索引擎数据采集与分析

```yaml
感知维度:
  AI搜索引擎:
    - ChatGPT Search: 搜索行为、引用来源、推荐逻辑
    - Perplexity: 引用偏好、答案结构、溯源模式
    - Google Gemini: 多模态搜索、知识图谱关联
    - 百度文心一言: 国内搜索生态、企业信息收录
    - 夸克/秘塔: 垂直搜索场景

  传统搜索引擎:
    - Google Search Console: 核心词排名、点击率、展现量
    - 百度站长平台: 国内SEO数据、索引状态
    - Bing Webmaster: 国际化搜索数据

  社交媒体:
    - 微信公众号: 文章阅读、分享、搜一搜排名
    - 小红书: 笔记曝光、搜索发现、关键词覆盖
    - 抖音: 视频搜索流量、话题热度
    - 知乎: 回答排名、专栏推荐

  竞品动态:
    - 竞品AI引用情况
    - 竞品内容策略变化
    - 行业GEO趋势

data_pipeline:
  采集层:
    - API对接（GSC、百度站长）
    - AI搜索模拟查询
    - 社媒数据抓取
    - 竞品监控爬虫
  
  解析层:
    - AI搜索意图分类（信息型/导航型/交易型）
    - 引用源权威度评分
    - 内容缺口识别
    - 语义聚类分析

输出格式:
  - GEO机会矩阵
  - 关键词AI引用率报告
  - 内容竞争态势图
  - 趋势预警信号
```

## 二、分析决策与策略规划层（大脑）

### 智能体2.1: GEO策略架构师 - `geo_strategist`

**职责**: 基于感知数据制定GEO策略，类似参考项目的"宪法约束决策引擎"

```yaml
策略框架 (GEO Constitution):
  合规约束:
    - 遵守各平台内容规范
    - 不制造虚假信息
    - 尊重知识产权
    - E-E-A-T原则（经验-专业-权威-可信）

  质量约束:
    - 内容原创率 ≥ 85%
    - 信息准确率 ≥ 95%
    - 引用来源权威性 ≥ 高
    - 用户满意度 ≥ 4.5/5

  效率约束:
    - 单篇内容生产时间 ≤ 2小时
    - 多平台分发时效 ≤ 24小时
    - AI引用目标提升 ≥ 30%/季度

决策场景:
  关键词策略:
    - 输入: 搜索量、竞争度、AI引用潜力、商业价值
    - 输出: 优先级排序、内容类型建议、目标平台

  内容策略:
    - 输入: 用户痛点、竞品内容、AI偏好
    - 输出: 内容大纲、结构化数据建议、引用优化方案

  平台策略:
    - 输入: 各平台规则、用户画像、算法特征
    - 输出: 平台适配方案、分发节奏、互动策略
```

### 智能体2.2: 内容重构工程师 - `content_engineer`

**职责**: 将传统内容转化为AI友好的结构化内容

```yaml
核心能力:
  结构化改造:
    - FAQ Schema标记
    - How-To步骤化
    - 数据表格化
    - 定义清单化
    - 对比矩阵化

  AI语义优化:
    - 实体明确标注
    - 因果关系清晰化
    - 结论前置化
    - 可引用片段设计

  多模态适配:
    - 文本（核心）
    - 图表（数据可视化）
    - 视频（演示类）
    - 音频（播客类）
```

## 三、内容生产与信源部署层（手脚）

### 智能体3.1: 信源部署专员 - `source_deployer`

**职责**: 在全网建立可被AI引用的权威信源节点

```yaml
部署渠道:
  官方阵地:
    - 官网SEO优化（技术SEO + 内容SEO）
    - 官方博客/新闻中心
    - 帮助中心/知识库
    - 开发者文档/API文档

  第三方阵地:
    - 行业媒体投稿
    - 知乎/百家号专栏
    - GitHub开源项目
    - 学术/白皮书发布
    - Wikipedia编辑（合规前提下）

  社媒阵地:
    - 微信公众号（长文深度）
    - 小红书（种草+教程）
    - 抖音（短视频科普）
    - B站（长视频讲解）

部署策略:
  1. 核心信源（官网）→ 权威性建设
  2. 放射信源（第三方）→ 扩散引用
  3. 社交信源（社媒）→ 互动传播
  4. 交叉引用 → 形成信源网络
```

### 智能体3.2: 效果监测分析师 - `geo_analyst`

**职责**: 监测GEO效果，形成闭环反馈

```yaml
监测指标:
  AI引用指标:
    - ChatGPT/Perplexity 引用次数
    - 引用场景（直接/间接）
    - 引用内容准确度
    - 品牌提及频率

  流量指标:
    - AI搜索带来的流量
    - 各平台搜索排名
    - 点击率(CTR) / 展现量
    - 转化率

  内容指标:
    - 内容被引用率
    - 内容完整性评分
    - E-E-A-T得分
    - 用户互动数据

报告输出:
  - 每日GEO简报
  - 周度趋势分析
  - 月度策略调整建议
  - 季度ROI报告
```

```yaml
    - 版权侵权检查
    - 敏感词过滤

  平台合规:
    - 各平台规则跟踪
    - 算法变化预警
    - 封号风险预判

  法律合规:
    - 广告法合规
    - 数据隐私保护

  - 负面舆情监测
```

**职责**: 持续学习AI搜索算法变化，自动优化策略

```yaml

    - 成功模式提取

    - 最佳实践沉淀
    - 自动策略调优

知识沉淀:
  - GEO战术库
    - 行业案例库
    - 模板素材库
    - 问答对知识库
```

```
用户需求输入
     │
     ▼
┌─────────────────────────────────────────┐
│         感知层 (geo_scout)              │
│  数据采集 → 意图分析 → 机会识别          │
└──────────────┬──────────────────────────┘
               │ 感知报告
               ▼
┌─────────────────────────────────────────┐
│       决策层 (geo_strategist)           │
│  策略制定 → 关键词规划 → 内容规划         │
│       (content_engineer 协同)            │
└──────────────┬──────────────────────────┘
               │ 策略方案
               ▼
┌─────────────────────────────────────────┐
│       执行层 (source_deployer)           │
│  内容生产 → 多平台部署 → 信源网络构建      │
└──────────────┬──────────────────────────┘
               │ 部署完成
               ▼
┌─────────────────────────────────────────┐
│       监控层 (geo_analyst)              │
│  效果监测 → 数据分析 → 报告生成           │
└──────────────┬──────────────────────────┘
               │ 分析报告
               ▼
┌─────────────────────────────────────────┐
│   风控层 (compliance_officer)            │
│  合规审核 → 风险评估 → 应急响应           │
└──────────────┬──────────────────────────┘
               │ 风控通过
               ▼
┌─────────────────────────────────────────┐
│       进化层 (geo_evolver)              │
│  策略优化 → 模式学习 → 自动迭代           │
└─────────────────────────────────────────┘
```

- **CMO团队**: 协同进行品牌营销和内容分发
