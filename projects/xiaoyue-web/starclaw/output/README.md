# 开源大模型异常日志分析评测指南

## 📋 评测流程

### 第一步：配置API Keys

在 `.env` 文件中添加以下配置：

```env
# 智谱AI（GLM-5-Code）
ZHIPU_API_KEY=your_zhipu_api_key

# 阿里云（Qwen2.5-Coder-32B）
ALIYUN_API_KEY=your_aliyun_api_key

# DeepSeek（DeepSeek-Coder-33B）
DEEPSEEK_API_KEY=your_deepseek_api_key

# OpenRouter（Code Llama、StarCoder2）
OPENROUTER_API_KEY=your_openrouter_api_key
```

### 第二步：运行评测脚本

```bash
cd xiaoyue-web/starclaw/output
node run_evaluation.js
```

### 第三步：查看结果

评测完成后，会生成以下文件：

- `evaluation_results.json` - 所有模型的详细输出
- `screenshots/` - 手动截取的模型输出截图

### 第四步：人工评分

根据模型输出，按照以下标准进行评分：

#### 评分维度（总分25分）

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **错误识别准确率** | 25% (5分) | 是否正确识别异常类型和关键字段 |
| **根因分析深度** | 30% (7.5分) | 分析是否触及根本原因（1-5分制×1.5） |
| **修复建议可执行性** | 25% (6.25分) | 建议是否具体、可操作、有效 |
| **响应效率** | 10% (2.5分) | Token消耗与响应时间的平衡 |
| **中文理解能力** | 10% (2.5分) | 中文日志的语义理解与表达流畅度 |

#### 根因分析深度评分细则

- **1分**：仅复述错误信息
- **2分**：识别直接原因
- **3分**：找到关联因素
- **4分**：触及根本原因
- **5分**：深入底层原理并给出预防措施

### 第五步：截图保存

对每个测试用例的每个模型输出，截取完整响应：

## 🔧 故障排查

### API调用失败

1. **检查API Key是否正确**
2. **检查网络连接**
3. **检查账户余额**

### 响应超时

1. **增加timeout参数**（默认60秒）
2. **使用代理或VPN**（国外模型）

### API限流

1. **增加请求间隔**（默认1秒）
2. **分批执行测试**
