# 测试用例 TC001：日志异常检测

## 测试目标
验证 StarClaw 在日志异常检测场景下的多模型表现

## 测试输入
```
[2026-03-18 09:23:15] ERROR: Database connection timeout
[2026-03-18 09:23:16] WARN: Retrying connection... attempt 1/3
[2026-03-18 09:23:18] ERROR: Connection failed after 3 attempts
[2026-03-18 09:23:19] ERROR: Service unhealthy, initiating failover
```

## 测试步骤
1. 切换至方舟 Coding 模型
2. 输入：`分析这段日志，找出异常原因和解决方案`
3. 记录响应时间和输出质量
4. 切换至智谱 AI 模型重复测试
5. 对比两个模型的分析结果

## 预期结果
- 方舟 Coding：给出代码级别的修复建议（连接池配置、超时设置）
- 智谱 AI：结构化分析，中文解释清晰

## 评分维度
| 维度 | 权重 | 标准 |
|------|------|------|
| 异常识别准确率 | 30% | 正确识别所有 ERROR 和 WARN |
| 根因分析深度 | 25% | 定位到数据库连接问题 |
| 建议可操作性 | 20% | 提供具体配置参数 |
| 响应速度 | 15% | < 3秒 |
| Token 效率 | 10% | 输出简洁有效 |

## 实测记录
| 模型 | 准确率 | 深度 | 可操作性 | 速度 | 总分 |
|------|-------|------|---------|------|------|
| 方舟 Coding | - | - | - | - | - |
| 智谱 AI | - | - | - | - | - |
